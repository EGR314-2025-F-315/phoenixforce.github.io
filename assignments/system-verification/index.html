System Verification

Team System Verification update for Phoenix Force · TerraGuard Rover — EGR 314 (Fall 2025). How our two-PCB system communicates, reacts to sensor data, and what each teammate delivered.

Wireless AP

Communication Method

Dedicated AP network hosted by the rover; the screen connects as a station. Traffic uses HTTP GET over Wi‑Fi—no ESP-NOW or ESP-Mesh.

Rover AP (ESP32-WROOM-32U)

  • SSID: TerraGuard-AP
  • Password: phoenixforce
  • IP: 192.168.4.1
  • Serves /metrics (JSON) and /stream (MJPEG)

Screen Module (ESP32-S3)

  • Joins AP as STA
  • Polls rover via periodic HTTP GET
  • Receives metrics + MJPEG frames
  • Two-way flow: continuous client requests + rover streams
Sensor → Output

Feedback-Controlled Responses

Rover PCB

Sensors:

  • BME688 (air quality: temp, humidity, gas resistance)
  • MQ-2 Gas & Smoke sensor
  • GPS (NEO-6M)
  • Camera (stream)

Behavior:

  • Gas readings → classify (clean, VOCs, smoke/CO, methane, butane, danger)
  • Air-quality state drives LED pattern logic
  • AP loss → hot-pink LED state

Screen PCB

Local outputs driven by rover data:

  • Not connected to AP → LED strip shows pink pattern
  • Good air → green LEDs
  • Moderate/warning → yellow LEDs
  • Poor air/danger → red LEDs
  • Dangerous gas or AP loss → speaker alert

Satisfies sensor input to actuator output requirement.

Outputs

Actuators Demonstrated

Rover PCB

  • WS2812B LED strip (status/air-quality)
  • Motor driver
  • GPS module outputs location telemetry

Screen PCB

  • LED strip with color-coded feedback
  • Speaker alerts for danger/AP loss
Power

Power Distribution

Only the rover PCB contains shared power rails; the screen runs on its own battery.

Rover Power System

  • Battery pack → charger → 5V regulator → 3.3V regulator
  • Rails power ESP32-WROOM-32U, BME688, MQ-2, GPS, LED strip, motor driver
  • 3.3V sensor rail and 5V LED rail demonstrable via multimeter

Screen Power

  • Local battery only; no shared rails with rover
  • Excluded from subsystem power checks
Final PCB

Subsystems Present on the Rover

ESP32-WROOM-32U Communication Core

Hosts AP, metrics endpoint, and camera streaming server.

BME688 Environmental Sensor

Temperature, humidity, gas resistance, and air-quality data.

MQ-2 Gas & Smoke Sensor

Detects smoke, CO, methane, propane/butane.

GPS Subsystem (NEO-6M)

Provides location and satellite data.

Motor Driver Subsystem

Part of the mechanical drive system (Erjan).

Power Regulation Subsystem

Battery, charger, 5V regulator, 3.3V regulator.
