Reflection

Lessons & Version 2.0

What worked, what we’d tune next time, and how we’d evolve the TerraGuard architecture.

Top 10

Lessons learned

  1. One toolchain per PCB saves sanity; mixing KiCad/Altium slowed merges.
  2. Subsystems worked alone but integration time was the real schedule driver.
  3. HTTP pull loop was simpler to debug than ESP-NOW for two-node comms.
  4. WS2812 feedback on both boards was the fastest way to see air-quality states.
  5. Always budget PSRAM and frame buffers early when planning MJPEG streaming.
  6. Pin planning needs to happen before routing—RF + camera lanes compete fast.
  7. Calibration for MQ-2 matters more than wiring; we locked in Ro before demos.
  8. Power budget reviews caught headroom gaps before we heated traces.
  9. Poster/storytelling benefits from early asset collection; it avoids last-minute design crunch.
  10. Documenting endpoints and message formats prevented “mystery 500s” on the HMI.
Top 5

Recommendations for future teams

  1. Pick one ECAD stack early and stick with it; enforce common footprints.
  2. Write a tiny loopback server to simulate the rover so your HMI can be developed offline.
  3. Lock down pin maps and regulators before any schematic review.
  4. Automate flashing + basic self-tests so every merge proves hardware still responds.
  5. Keep a single source of truth for endpoints and message shapes; update it when you change code.
Next

Version 2.0 ideas

Communication & software

  • Add a command channel from screen → rover (basic drive + OTA).
  • Move to protobuf/CBOR to shrink payloads and add versioning.
  • Stream pipeline with double-buffering + checksum per frame for resilience.
  • Better health pings so the HMI can show link quality and jitter.
  • Split code into comms, sensors, UI, and drivers to isolate bugs faster.

Hardware

  • Thicker power pours and shorter LED traces based on feedback.
  • Shield the RF path and give GPS a cleaner keep-out.
  • Add current sensing to validate the power budget on-board.
  • Bring test pads for UART/JTAG to speed up field debugging.
  • Drop-in connector for a future particulate sensor or CO sensor.
