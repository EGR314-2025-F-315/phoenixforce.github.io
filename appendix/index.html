Appendix A - Organization

Supporting Material

Extra detail that supports our Team Organization page-notes, optional steps, and working artifacts that don’t fit in the main body.

Notes & Brainstorms

  • Assumptions: rugged embedded device for wildfire response; glove-friendly screen+buttons; internet-optional comms (ESP-NOW/LoRa); outdoor sun/dust.
  • Key metrics: ≤1 s alert latency; 99% message delivery (n≥100 in lab); >95% detection accuracy for chosen condition; ≤3 actions for core task.
  • Constraints: course timeline, limited budget, safe lab surrogates (no open flame), simple enclosure & power budget.
  • Why this scope: maximizes embedded-systems learning while remaining demo-feasible; clear pass/fail tests.
  • Deliverables to outlive class: repo with build notes, wiring/schematic, BOM, 2-min demo video, one-pager.

Optional Team Organization Steps

  • Expectations Charter: response time ≤24h on chat; meetings start on time; escalate blockers after 2 days.
  • Feedback Cadence: 5-minute retro at end of weekly checkpoint (keep/change/try next).
  • Conflict Path: 1:1 → team discussion (15 min timebox) → instructor input if unresolved.
  • Peer Reviews: every PR needs one review; use a 3-item checklist (builds clean, tests noted, docs touched).
  • Role Rotation (optional): midpoint swap of a small subtask to cross-train (e.g., HMI helps with comms retries).

Definition of Done (DoD)

  • Builds from a clean clone; README updated.
  • Test case(s) executed with logged results and short clip.
  • Wiring/schematic and enclosure notes updated.
  • Metrics recorded (latency, delivery, accuracy as applicable).

Milestones (subject to course calendar)

  • Week 2: choose sensors & comms; trade studies started.
  • Week 4: vertical slice (sensor → alert → message).
  • Week 6: usability pass (gloves/sun); tutorial added.
  • Week 8: reliability run (n≥100 messages; 99% success).
  • Final: polished demo + portfolio package.

Test Plan Summary

  • Latency: timestamp sensor event → alert display → message ack.
  • Reliability: send 100+ messages under varied distance/orientation; log success/RTT.
  • Detection: lab surrogate stimuli; record TP/FP/FN/TN; plot confusion matrix.
  • Usability: task time for core flow; 60-sec learnability check.
